When comparing Git pull vs fetch, Git fetch is a safer
alternative because it pulls in all the commits from your remote
but doesn’t make any changes to your local files. On the other
hand, Git pull is faster as you’re performing multiple actions in
one – a better bang for your buck. Using the Git pull command can be
seen in one light as a feature of convenience; you’re probably less
worried about introducing conflicts into your local repo and you just
want the most up-to-date changes from the remote branch you’re pulling
from. Git pull is a more advanced action and it’s important to
understand that you will be introducing changes and immediately
applying them to your currently checked out branch.
